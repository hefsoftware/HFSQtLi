HFSQtLi 0.0.1
A Qt/C++ wrapper for SQLite3.
Build instruction and requirements

HFSQtLi requires Qt as well as SQLite. For more information on Qt please consult http://www.qt.io and for SQLite please see https://sqlite.org/. The library was tested with Qt 6.2.2 and SQLite 3.37.1. It does not provide the Qt library (which should be downloaded and installed separately) nor SQLite library which amalgamation should be downloaded and put in Test and/or HFSQtLi folders.

The library developement is done with the project found in Test. When the library is considered stable it is possible to run the script Test/amalgamate.py. This will parse the headers and source files and generate one header (HFSQtLi/HFSQtLi.h) and one header (HFSQtLi/HFSQtLi.cpp). Those files can be included in any external project to give access to the full functionality of the library.

The source should be compiled with SQLITE_ENABLE_COLUMN_METADATA to enable the automatic binding of blobs. Without it a warning will be issued when compiling. The library will still fully work except expression such as:

Blob b; // Note that auto-open is enabled by default
m_db->executeSingle("SELECT blobColumn, rowid FROM TestTable WHERE name="Foo", b, b);
